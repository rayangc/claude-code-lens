## Tasks

### Task 1: Keyboard Shortcuts for Thinking + Tool Output Toggle
**Agent: ux-agent**

Add global keyboard shortcuts to the session detail view:
- **Ctrl+T**: Toggle ALL thinking blocks expanded/collapsed at once
- **Ctrl+O**: Toggle ALL tool call output sections expanded/collapsed at once
- **Esc**: Reset both to default state (thinking collapsed, tool outputs collapsed)

Implementation:
- Add state to session detail page: `thinkingExpanded: boolean`, `toolOutputsExpanded: boolean`
- Pass these as props to MessageContent → ThinkingBlock and ToolCallBlock
- ThinkingBlock: respect `forceExpanded` prop (overrides individual toggle). **Global overrides all** — when global state is active, individual toggles are disabled. Esc resets to allow individual control.
- ToolCallBlock: respect `forceOutputExpanded` prop for the output section (same override behavior)
- Add a **sticky** keyboard shortcut hint bar at the very top of the session view (always visible, like cookbook):
  `"Ctrl+T toggle thinking · Ctrl+O toggle tool outputs · Esc Reset"`
- Use `useEffect` with `keydown` listener for the shortcuts

**Reference**: See `screenshots/thinking-*.png` and `screenshots/tool-call-*.png`

### Task 2: Filter Left Rail Only (Not Main Content)
**Agent: ux-agent**

Change the All/Conv/User filter behavior:
- Currently: filters affect BOTH the message tree AND main content
- New behavior: filters affect ONLY the left rail message tree. Main content always shows all messages.
- This makes the left rail a navigation filter — click a tree entry to scroll to that message in the full conversation

Also add a text "filter..." input above the tree (like the cookbook has) that filters tree entries by **all content** (user messages, assistant text, tool names, and tool output text).

When clicking a tree entry to scroll to a message, **briefly highlight** the target message (subtle border glow or bg pulse for ~1 second) so it's obvious which one was navigated to.

**Files**: SessionNav.tsx, MessageTree.tsx, MessageContent.tsx, session/[id]/page.tsx

**Reference**: See `screenshots/filter-*.png`

### Task 3: Message Background Colors by Role
**Agent: design-agent**

Update message backgrounds to differentiate by role:
- **User messages**: Distinct dark background (e.g., near-black `#0d0d0d` or similar)
- **Tool call blocks**: Subtle green-tinted background (very muted, not distracting)
- **Assistant text + thinking**: Default page background (no special bg)

Use the `/frontend-design` skill to make the exact color choices. The goal is clear visual separation without being garish.

**Files**: MessageContent.tsx (user message bg), ToolCallBlock.tsx (tool call bg), ThinkingBlock.tsx (ensure default bg)

**Reference**: See cookbook screenshots for how they handle this

### Task 4: Flatten Left Rail Nesting to Max 2 Levels
**Agent: ux-agent**

The current message tree is deeply nested (see `screenshots/nest-3-*.png` — 8+ indent levels). Flatten to max 2 levels like the cookbook:

**Level 1**: User messages (with text preview)
**Level 2**: Under each user message, flat list of:
  - Assistant text preview (truncated)
  - Tool call names with short context preview (e.g., "Read — src/lib/types.ts", "Bash — npm run build"). Click scrolls to the tool call in main content.

No deeper nesting. Tool calls under an assistant response are listed flat, not indented further. This matches the cookbook pattern (see `screenshots/nest-1-*.png`).

**Files**: MessageTree.tsx (restructure the tree rendering)

### Task 5: Mobile Responsive — Hamburger Menu for Sidebar
**Agent: mobile-agent**

On mobile/small screens, the left sidebar overlaps with content and can't be hidden. Fix:

- Add a hamburger menu button (☰) in the top-left corner, visible only on small screens (`md:hidden`)
- On mobile: sidebar is hidden by default, hamburger toggles it as a full-width overlay
- On desktop: sidebar is always visible (current behavior)
- When sidebar is open on mobile, tapping outside or clicking a tree entry closes it
- Apply to BOTH the dashboard (ProjectSidebar) and session detail (SessionNav)

**Files**: session/[id]/page.tsx, page.tsx, SessionNav.tsx, ProjectSidebar.tsx

**Reference**: See `screenshots/mobile-*.PNG`
