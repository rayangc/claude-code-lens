# V1.4 — Mobile Scroll Fix, Left Rail Cleanup, Team Sessions

## Overview

Three changes: fix mobile horizontal scrolling, clean up left rail dots, and add team session rendering support.

---

## 1. Mobile Horizontal Scroll Fix

**Problem**: On mobile, the page can be dragged left/right, revealing empty space. Content overflows the viewport horizontally.

**Fix approach**: Belt-and-suspenders.

- Add `overflow-x: hidden` on `html` and `body` elements (global catch)
- Audit and fix individual containers that leak (long file paths, code blocks, pre elements)
- Ensure `word-break: break-word` or `overflow-wrap: break-word` on text containers
- Add `max-width: 100vw` as a safety net on the outermost layout wrapper
- Test both dashboard and session detail views on mobile viewport

**Files likely affected**: `src/app/globals.css`, `src/app/layout.tsx`, `src/app/session/[id]/page.tsx`, `src/components/session/MessageContent.tsx`, `src/components/session/ToolCallBlock.tsx`

---

## 2. Remove Left Rail Dots

**Problem**: Blue and purple dots appear before "User" and "Assistant" labels in the session left rail navigation tree. They add visual noise without value.

**Fix**: Remove the colored dot indicators entirely. No replacement — the role label text is sufficient.

**Files affected**: `src/components/session/MessageTree.tsx`

---

## 3. Team Session Support

### Detection

A session is a "team session" if any user-role message contains `<teammate-message teammate_id="...">` XML tags.

**Detection method**: Scan user message content for the `<teammate-message` pattern. Extract unique `teammate_id` values.

### Dashboard Changes

- On session cards in the dashboard, if a session is a team session:
  - Show a small "team" badge/indicator
  - List the unique teammate names found in the session

**Data flow**: The session metadata (from `/api/sessions`) should include `isTeamSession: boolean` and `teammateNames: string[]`. This requires parsing teammate_id values during session scanning.

**Files affected**: `src/lib/parser/jsonl.ts` (extract teammate info), `src/lib/types.ts` (add fields), `src/components/dashboard/SessionCard.tsx`, API routes as needed.

### Session Detail View — Label Replacement

- When a user-role message contains `<teammate-message teammate_id="X">`, replace the "User" label with the value of `teammate_id` (e.g., "team-lead", "researcher", "backend")
- Apply in both the left rail tree and the main content area message header

### Session Detail View — Content Rendering

**Text messages** (task assignments, instructions):
- Strip the `<teammate-message>` XML wrapper tags entirely
- Show the inner text content as plain text (not markdown-rendered)
- Show in full — no truncation, even for long task assignments
- The teammate_id becomes the message label

**JSON protocol messages** (shutdown_request, shutdown_approved, idle, task_completed):
- Detect by checking if the inner content of `<teammate-message>` is valid JSON with a `type` field
- Render as compact status pills in the main content area:
  - `shutdown_request` → red dot + "Shutdown requested" + reason if present
  - `shutdown_approved` → gray square + "Shutdown approved"
  - `idle` → yellow dot + "Idle"
  - Other JSON types → show `type` value as a muted label
- In the left rail tree: show as compact single-line entry (e.g., "[stopped] approved")

### Parsing Logic

Add a utility to parse `<teammate-message>` content:

```
Input: raw message text containing <teammate-message teammate_id="X">content</teammate-message>
Output: { teammateId: string, content: string, isJson: boolean, jsonType?: string, jsonData?: object }
```

Use regex to extract — no need for a full XML parser. Pattern: `<teammate-message\s+teammate_id="([^"]+)"[^>]*>([\s\S]*?)<\/teammate-message>`

### Left Rail Tree for Team Messages

- Protocol JSON messages: show as compact entry like "[stopped] approved"
- Text messages: show normally with teammate name as label instead of "User"
