## Tasks

### Task 1: Fix Mobile Scrolling
**Agent: mobile-agent**

On mobile, the page scrolling is broken — content isn't anchored properly when scrolling. The likely cause is the `h-screen` + `overflow` setup on the outermost flex container conflicting with the mobile sidebar overlay (fixed positioning + transforms).

Investigation needed:
- The session detail page has `flex h-screen` on the outer div
- The sidebar uses `fixed inset-y-0` positioning on mobile
- The main content area needs to scroll independently
- The hamburger button is `fixed top-3 left-3 z-50`

Likely fixes:
- Ensure the main content div has `overflow-y-auto` and fills the remaining space
- On mobile, the outer container should not have `overflow-hidden`
- The fixed sidebar overlay should not affect the main content scroll position
- Test by scrolling up/down on mobile viewport — content should stay anchored

**Files**: `src/app/session/[id]/page.tsx`, `src/app/page.tsx`

### Task 2: Fix Left Rail Hierarchy
**Agent: tree-agent**

Current: User messages at level 1, everything else (assistant + tool calls) flat at level 2.
Target (matching cookbook): User AND assistant turns at the SAME level, with tool calls nested under assistant.

**Cookbook pattern** (from screenshots):
```
user: how many workouts have I done
  assistant: Let me check your workout logs...
    ⚙ Glob
    ⚙ Read
  assistant: So right now I only have e+1 workouts...
user: I'm having trouble with...
  assistant: Now let me extract frames to analyze th...
    ⚙ Bash
    ⚙ Read
    ⚙ Read
```

Key changes to `MessageTree.tsx`:
- Level 1: Both `user:` and `assistant:` entries with text previews, labeled with role prefix
- Level 2: Tool calls nested UNDER their parent assistant message (not flat with assistant text)
- Add `user:` and `assistant:` prefix labels to tree entries
- User entries: blue color, `user:` prefix
- Assistant entries: purple color, `assistant:` prefix
- Tool call entries: muted/tertiary, `⚙` prefix with tool name + context preview

The grouping logic: walk messages in order. Each user message starts a new group. Assistant messages within that group are at level 1 (same as user). Tool calls under each assistant are at level 2.

**Files**: `src/components/session/MessageTree.tsx`

### Task 3: Global Stats on Dashboard Landing
**Agent: stats-agent**

Add a `GET /api/stats` endpoint and update the dashboard empty state to show hero stats.

#### API Endpoint: `GET /api/stats`

Returns:
```json
{
  "totalSessions": 847,
  "totalProjects": 23,
  "totalInputTokens": 12400000,
  "totalOutputTokens": 8200000
}
```

Implementation:
- Scan all project directories (reuse scanner logic)
- For each session JSONL file, parse events and sum `message.usage.input_tokens` and `message.usage.output_tokens` from all events
- Count total sessions and projects
- Return aggregated stats
- No caching for now — compute fresh each request
